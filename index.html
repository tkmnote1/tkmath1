2. Đại số tuyến tính

VectorMatrix

2.1. Vector

Vector có thể hiểu là một đối tượng có độ lớn và hướng. Trong đại số tuyến tính, vector thường được biểu diễn dưới dạng cột hoặc hàng.

v = (v1, v2, ..., vn)
Khái niệm Ý nghĩa Ví dụ
Vector không Vector có tất cả tọa độ bằng 0. (0, 0, 0)
Tổ hợp tuyến tính Kết hợp các vector bằng phép nhân vô hướng và cộng. u = 2v1 - 3v2
Độc lập tuyến tính Không vector nào biểu diễn được qua các vector còn lại. Hai vector không cùng phương trong R².

2.2. Ma trận

Ma trận là bảng số gồm hàng và cột. Trong lập trình, ma trận có thể được lưu bằng mảng hai chiều. Trong toán, ma trận biểu diễn phép biến đổi tuyến tính hoặc hệ phương trình tuyến tính.

A = [aij]m×n

Các phép toán cơ bản

  • Cộng ma trận: hai ma trận phải cùng kích thước.
  • Nhân ma trận với số: nhân từng phần tử với số đó.
  • Nhân hai ma trận: nếu A có kích thước m×n và B có kích thước n×p thì AB có kích thước m×p.
  • Ma trận chuyển vị: đổi hàng thành cột.
// Cộng hai ma trận A và B kích thước m x n trong C
for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++) {
        C[i][j] = A[i][j] + B[i][j];
    }
}

2.3. Hệ phương trình tuyến tính

Hệ tuyến tính thường được viết dưới dạng:

Ax = b

Trong đó A là ma trận hệ số, x là vector ẩn, b là vector vế phải. Bài toán là tìm x sao cho phương trình đúng.

2.4. Trị riêng và vector riêng

Nếu A là ma trận vuông, một vector khác 0 gọi là vector riêng của A nếu sau khi nhân với A, nó chỉ bị co giãn chứ không đổi hướng.

Av = λv, với v ≠ 0
Ý nghĩa: trị riêng cho biết hệ số co giãn theo một hướng đặc biệt, vector riêng cho biết hướng đặc biệt đó.

2.5. Không gian Euclid và hình chiếu

Trong Rn với tích vô hướng chính tắc, hình chiếu trực giao của vector u lên không gian con W là vector thuộc W gần u nhất.

d(u, W) = ||u - prW(u)||

2.6. Dạng toàn phương

Dạng toàn phương thường có dạng:

Q(x) = xT A x

Nếu A đối xứng, ta có thể chéo hóa trực giao để đưa Q về dạng chính tắc. Dạng toàn phương dùng để xét dấu, cực trị và hình học bậc hai.

3. Giải tích

LimitDerivative

3.1. Giới hạn

Giới hạn mô tả giá trị mà hàm số tiến tới khi biến tiến tới một điểm nào đó. Đây là nền tảng để định nghĩa đạo hàm và tích phân.

limx→a f(x) = L

3.2. Đạo hàm

Đạo hàm đo tốc độ thay đổi tức thời của hàm số. Về hình học, đạo hàm là hệ số góc của tiếp tuyến.

f'(x) = limh→0 [f(x+h)-f(x)] / h
Hàm số Đạo hàm
xn n xn-1
sin x cos x
cos x -sin x
ex ex
ln x 1/x

3.3. Đạo hàm riêng

Với hàm nhiều biến f(x, y), đạo hàm riêng theo x là đạo hàm khi xem y là hằng số.

∂f/∂x, ∂f/∂y

3.4. Cực trị nhiều biến

Điểm dừng của f(x, y) thỏa:

fx(x, y) = 0 và fy(x, y) = 0

Sau đó dùng Hessian để phân loại cực trị:

H = [[fxx, fxy], [fyx, fyy]]
Điều kiện Kết luận
det(H) > 0 và fxx > 0 Cực tiểu địa phương
det(H) > 0 và fxx < 0 Cực đại địa phương
det(H) < 0 Điểm yên ngựa

3.5. Tích phân

Tích phân bất định dùng để tìm nguyên hàm. Tích phân xác định dùng để tính diện tích có dấu, tổng tích lũy hoặc đại lượng vật lý.

∫ f(x) dx = F(x) + C, với F'(x) = f(x)

4. Toán rời rạc

LogicGraph

4.1. Mệnh đề logic

Mệnh đề là câu khẳng định có giá trị đúng hoặc sai. Logic mệnh đề là nền tảng của lập trình điều kiện.

Ký hiệu Ý nghĩa Liên hệ lập trình
¬P Phủ định P !P
P ∧ Q P và Q P && Q
P ∨ Q P hoặc Q P || Q
P ⇒ Q Nếu P thì Q if (P) then Q

4.2. Quy nạp toán học

Quy nạp dùng để chứng minh mệnh đề đúng với mọi số tự nhiên.

  1. Chứng minh mệnh đề đúng với n = n0.
  2. Giả sử đúng với n = k.
  3. Chứng minh đúng với n = k + 1.

4.3. Tổ hợp

P(n, k) = n! / (n-k)!
C(n, k) = n! / [k!(n-k)!]

4.4. Đồ thị

Đồ thị gồm tập đỉnh và tập cạnh. Trong lập trình, đồ thị dùng để mô hình hóa mạng xã hội, bản đồ đường đi, quan hệ phụ thuộc, cây thư mục.

  • Đồ thị vô hướng: cạnh không có chiều.
  • Đồ thị có hướng: cạnh có chiều.
  • Đường đi: dãy đỉnh liên tiếp nhau thông qua cạnh.
  • Chu trình: đường đi bắt đầu và kết thúc ở cùng một đỉnh.

6. Phương trình vi phân

ODESystem

6.1. Phương trình vi phân là gì?

Phương trình vi phân là phương trình chứa hàm chưa biết và đạo hàm của nó. Nó dùng để mô hình hóa chuyển động, tăng trưởng, dao động, mạch điện, lan truyền nhiệt.

6.2. Hệ tuyến tính không thuần nhất

Dạng tổng quát:

y'(x) = A(x)y(x) + g(x)

6.3. Phương pháp biến thiên tham số

  1. Giải hệ thuần nhất: yh = Y(x)c.
  2. Lập ma trận nghiệm cơ bản Y(x) từ các nghiệm cột của hệ thuần nhất.
  3. Đặt nghiệm riêng: yp = Y(x)u(x).
  4. Suy ra Y(x)u'(x) = g(x), nên u'(x) = Y(x)-1g(x).
  5. Tích phân để tìm u(x), rồi kết luận y = yh + yp.
Quy trình ghi lời giải:
1. Tìm nghiệm tổng quát hệ thuần nhất.
2. Chọn ma trận nghiệm cơ bản Y(x).
3. Đặt y_p = Y(x)u(x).
4. Giải Y(x)u'(x) = g(x).
5. Tích phân u'(x).
6. Kết luận nghiệm tổng quát.

11. Thuật toán

AlgorithmComplexity

11.1. Độ phức tạp thuật toán

Độ phức tạp giúp ước lượng thuật toán chạy nhanh hay chậm khi kích thước dữ liệu tăng.

Ký hiệu Ý nghĩa Ví dụ
O(1) Thời gian hằng số Truy cập a[i]
O(log n) Tăng rất chậm Tìm kiếm nhị phân
O(n) Duyệt qua n phần tử Tìm max trong mảng
O(n²) Hai vòng lặp lồng nhau Bubble sort

11.2. Tìm kiếm tuyến tính

int linearSearch(int a[], int n, int x) {
    for (int i = 0; i < n; i++) {
        if (a[i] == x) return i;
    }
    return -1;
}

11.3. Tìm kiếm nhị phân

Áp dụng khi mảng đã được sắp xếp. 

int binarySearch(int a[], int n, int x) {
    int left = 0, right = n - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (a[mid] == x) return mid;
        if (a[mid] < x) left = mid + 1;
        else right = mid - 1;
    }

    return -1;
}

11.4. Sắp xếp nổi bọt

void bubbleSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (a[j] > a[j + 1]) {
                int temp = a[j];
                a[j] = a[j + 1];
                a[j + 1] = temp;
            }
        }
    }
}

11.5. Đệ quy

Đệ quy là kỹ thuật hàm gọi lại chính nó. Một hàm đệ quy phải có điều kiện dừng. 

int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);
}
Cẩn thận: Nếu thiếu điều kiện dừng, chương trình có thể bị tràn stack.

Khi trình duyệt gọi đến server, nó gửi một request. Server xử lý và trả về response.

Thành phần Ý nghĩa
HTTP Method GET, POST, PUT, PATCH, DELETE.
URL Địa chỉ tài nguyên.
Headers Thông tin bổ sung như Content-Type, Authorization.
Body Dữ liệu gửi kèm request.

13. Cơ sở dữ liệu

SQLDatabase

13.1. Dữ liệu và thông tin

Dữ liệu là các giá trị thô. Thông tin là dữ liệu đã được xử lý và đặt trong ngữ cảnh có ý nghĩa.

13.2. Bảng, dòng, cột

  • Bảng: tập hợp dữ liệu cùng loại.
  • Dòng: một bản ghi cụ thể.
  • Cột: thuộc tính của bản ghi.

13.3. Khóa chính và khóa ngoại

Khái niệm Ý nghĩa
Primary Key Định danh duy nhất cho mỗi dòng trong bảng.
Foreign Key Liên kết tới khóa chính của bảng khác.

13.4. SQL cơ bản

-- Lấy toàn bộ sinh viên
SELECT * FROM Students;

-- Lọc sinh viên có điểm GPA >= 8
SELECT id, name, gpa
FROM Students
WHERE gpa >= 8;

-- Sắp xếp giảm dần theo GPA
SELECT *
FROM Students
ORDER BY gpa DESC;

13.5. JOIN

SELECT Students.name, Classes.class_name
FROM Students
JOIN Classes
ON Students.class_id = Classes.id;

14. Git & DevOps

GitDevOps

14.1. Git cơ bản

Git là hệ thống quản lý phiên bản mã nguồn. Nó giúp theo dõi lịch sử thay đổi và phối hợp làm việc nhóm. 

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/username/repository.git
git push -u origin main

14.2. Các khái niệm quan trọng

Khái niệm Ý nghĩa
Repository Kho chứa mã nguồn.
Commit Một mốc lưu thay đổi.
Branch Nhánh phát triển độc lập.
Merge Gộp nhánh.
Pull Request Đề xuất gộp code và review.

14.3. DevOps

DevOps kết hợp development và operations, nhằm tự động hóa quy trình build, test, deploy.

  • CI: tự động build và test khi có code mới.
  • CD: tự động triển khai ứng dụng.
  • Docker: đóng gói ứng dụng vào container.
  • Jenkins/GitHub Actions: công cụ tự động hóa pipeline.

16. Đưa file này lên GitHub Pages

GitHub PagesDeploy

16.1. Cách dùng nhanh

  1. Tạo repository mới trên GitHub, ví dụ knowledge-site.
  2. Tạo file index.html.
  3. Dán toàn bộ nội dung file này vào index.html.
  4. Commit changes.
  5. Vào Repository → Settings → Pages.
  6. Chọn Deploy from a branch.
  7. Chọn branch main và folder /root.
  8. Bấm Save và chờ link website xuất hiện.
